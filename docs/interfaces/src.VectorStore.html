    Interface VectorStore

    Defines the essential operations for a vector store. A vector store efficiently stores and retrieves high-dimensional vectors, facilitating similarity searches for AI applications like semantic search. It provides core functionalities for managing documents (add, update, delete) and performing similarity-based queries.

    interface VectorStore {
        load: () => Promise<VectorStore>;
        unload: () => Promise<void>;
        add(
            params: {
                documents: string[];
                embeddings?: number[][];
                ids?: string[];
                metadatas?: Record<string, any>[];
            },
        ): Promise<string[]>;
        delete(
            params: { ids?: string[]; predicate?: (value: GetResult) => boolean },
        ): Promise<void>;
        query(
            params: {
                ids?: string[];
                nResults?: number;
                predicate?: (value: QueryResult) => boolean;
                queryEmbeddings?: number[][];
                queryTexts?: string[];
            },
        ): Promise<QueryResult[][]>;
        update(
            params: {
                documents?: string[];
                embeddings?: number[][];
                ids: string[];
                metadatas?: Record<string, any>[];
            },
        ): Promise<void>;
    }

      Performs a similarity search against the stored vectors.

      Parameters

      • params: {
            ids?: string[];
            nResults?: number;
            predicate?: (value: QueryResult) => boolean;
            queryEmbeddings?: number[][];
            queryTexts?: string[];
        }

        Object containing:

        • queryTexts (optional): The raw query strings to search for.
        • queryEmbeddings (optional): Pre-computed embeddings for the queries.
        • nResults (optional): The number of top similar results to return per query.
        • ids (optional): Restrict the search to these document IDs.
        • predicate (optional): Function to filter results after retrieval.

      Returns Promise<QueryResult[][]>

      A promise that resolves to an array of result arrays (one per query).
